# Исследование: профили микрофонов (АЧХ / коррекция)

Цель — понять, **откуда брать кривые**, можно ли их **легально** положить в продукт, и **что делать в коде** в `MicCorrectionNode` (сейчас pass-through, см. `docs/ROADMAP.md`).

**Список из 30 микрофонов** (ориентир для профилей): см. [`docs/MIC_PROFILES_LIST.md`](MIC_PROFILES_LIST.md).

---

## Что есть в интернете (кратко)

### 1. Каталоги с **графиками** (часто картинки, не CSV)

| Ресурс | Что даёт | Экспорт чисел |
|--------|----------|----------------|
| **[microphone.wiki](https://microphone.wiki/database/)** | Много моделей, описания, интерактивный/картинка АЧХ, «tone scale» по зонам | Публичного массового **CSV/API** под коррекцию не видно — по сути **визуальный** каталог. |
| **[RecordingHacks.com](https://recordinghacks.com/)** | Классическая база: карточки микрофонов, часто график АЧХ, заметки | Обычно **изображение** графика из даташита или измерения, не готовый набор точек для импорта. |
| **Даташиты производителей** (Shure, Sennheiser, AKG, Behringer, Rode…) | Официальные PDF с кривой АЧХ | **Нужна оцифровка** графика (ручная или тулзой) → ваши **собственные** таблицы частота–дБ; права на **саму кривую** в даташите — типично у производителя; **повторное использование цифр**, полученных измерением/оцифровкой «как факт», обычно обсуждают с юристом при коммерческом продукте. |

### 2. Данные в **файлах** (удобнее для DSP)

| Ресурс | Формат | Ограничение |
|--------|--------|-------------|
| **[MicIRP](https://audeering.github.io/datasets/datasets/micirp.html)** (Microphone Impulse Response Project, Xaudia / Stewart Tavener) | **WAV** ИХ, 44.1/48 kHz, 24 bit, **CC-BY-SA-4.0**, коммерция разрешена | **66 винтажных** микрофонов, не «топ-30 современных вокальных»; к тому же это **ИХ в конкретной кабине**, не «идеальная АЧХ в свободном поле». Для продукта — годится как **один из источников** для режима «винтаж/IR», не как полный список SM58/BM800. |

### 3. Про «большой сайт и 2500»

Точного публичного «одного сайта с 2500 пользовательскими кривыми в CSV» в выдаче не нашлось. Часто люди путают с:

- крупными **каталогами** микрофонов (тысячи **карточек**), где **график** всё равно картинка/интерактив без выгрузки;
- базами **наушников/IEM** (Oratory1990, Squiglink и т.д.) — это **не микрофоны**;
- калибровочными базами под **измерительные** микрофоны (например, серийные кривые под конкретный RTA-микрофон), а не под SM58.

---

## Почему нельзя просто «скопировать 30 кривых с картинки»

1. **Точность**: с картинки без калиброванного замера получится грубая аппроксимация; для **коррекции** это может звучать хуже, чем без неё.  
2. **Право**: чужие **графики** и бренды — отдельный вопрос; **свои** таблицы, полученные из **своих** измерений или из **лицензированно** разрешённых датасетов — предпочтительнее.  
3. **Один микрофон — много кривых**: расстояние, ось, сетка даташита vs реальное положение — «одна кривая на модель» для студии часто **недостаточно** для hi-fi коррекции.

## Предлагаемый состав «~30 моделей» (ориентир по нишам)

Не числовые данные — только **список для закупки/измерений/поиска даташитов**. Разбивка: **бюджет / середина / дорого**, плюс **частые для вокала**.

**Очень дешёвые / «китай/стрим»**  
Behringer C-1, Behringer B-1, Behringer XM8500, «BM-800» класса, Samson C01, Fifine (популярные модели), Maono PM422 — при необходимости заменить на те, что реально продаются в вашем регионе.

**Народный сегмент**  
Shure SM58, SM48, PGA48 / PGA58, Sennheiser e835, e845, AKG D5, P120, P5, Rode NT1-A, Audio-Technica AT2020, sE X1.

**Середина / «студия»**  
Shure Beta 58A / Beta 87A, Rode NT1 (5th gen), AKG C214, Audio-Technica AT4040, sE 2200.

**Дороже (эталонные имена)**  
Neumann TLM 103, AKG C414 (варианты), Telefunken/Neumann/UAD-стиль моделей по выбору продуктовой линейки.

Итоговые **30** лучше **зафиксировать** в продукте одним списком после выбора регионального ассортимента.

## Следующий шаг в **коде** (реалистичный порядок)

1. **Формат данных** в репозитории: например JSON с полями `id`, `displayName`, `sampleRate`, и либо  
   - массив `{ "freqHz", "gainDb" }` (относительно выбранного референса), либо  
   - коэффициенты **IIR** / FIR-тап (если офлайн посчитали).  
2. **DSP**: интерполяция таблицы → каскад PEQ или короткий **FIR**; **amount** и **bypass** как уже заложено в `MicCorrectionNode`.  
3. **Источник кривых** (параллельно):  
   - для **винтажного** набора — рассмотреть **MicIRP** (лицензия CC-BY-SA-4.0, указать авторство);  
   - для **массового** списка — **свои** замеры или лицензированные данные, либо осторожная оцифровка даташитов с юридической проверкой.

Без этого шага «перенести АЧХ в плагин» = только **инфраструктура** без проверяемого звука.

---

## Резюме

- **Готовых** «30 самых популярных микрофонов в одном открытом CSV» для свободного коммерческого копирования **не найдено**; типичный путь — **даташиты + оцифровка**, **свои измерения**, или **лицензированные** датасеты (например **MicIRP** для подмножества винтажа).  
- Сайты вроде **microphone.wiki** и **RecordingHacks** — хороши для **ориентира и картинок**, не для автоматического импорта тысяч кривых.  
- **Следующий инженерный шаг**: зафиксировать **формат профиля** + минимальный **EQ/FIR** в `MicCorrectionNode`, затем наполнять **реальными** кривыми по одному источнику за раз.
