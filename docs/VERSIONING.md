# Версионирование (Razumov VocalChainer)

## До релиза 1.0

- Формат: **`0.9.x`** (семантика `MAJOR.MINOR.PATCH`, где сейчас **MINOR зафиксирован на 9** для предрелизной линии).
- **PATCH** растёт последовательно: `0.9.8`, `0.9.9`, `0.9.10`, `0.9.11`, ... — допустимы **двузначные и большие** значения patch (это не «десятичная дробь»).
- Не путать с прыжком на **`0.10.0`** без отдельного решения: это уже смена **MINOR**, не следующий шаг после `0.9.9`.
- Первый **публичный релиз** продукта планируется как **`1.0.0`** (отдельное решение, не автоматический переход с последней 0.9.x).

## Где зафиксирована версия

- `CMakeLists.txt`: `project(... VERSION x.y.z)` и `VERSION` внутри `juce_add_plugin` — держать **синхронно**.

## Связь с GitHub

- Репозиторий: **Razumov_VocalChainer**; локальная папка проекта может оставаться прежним именем каталога — на версию это не влияет.
