# Razumov Vocal Chain — продуктовое видение и архитектурный roadmap

Документ для **преемственности**: цели продукта, отличие от ShaperX, крупные этапы по DSP/UI/архитектуре. Детальная табличная очередь по-прежнему в **`docs/ROADMAP.md`**.

**Визуальный референс UI:** мокапы «Main / Advanced» (лента нод + инспектор снизу; опционально граф с браузером модулей и кабелями) — стиль boutique / тёмная студия; цветовые акценты по типу обработки (см. также `docs/design/MainView_blueprint.svg`).

---

## Порядок приоритетов (зафиксировано)

**Сейчас и до зрелого ядра — в первую очередь:**

- **Цепочка и граф:** сборка/изменение цепочки, инсерты, параллели, сохранение топологии, UI цепочки (Main / по мере готовности Advanced).
- **Макроконтроллеры:** расширение и маппинг (сейчас четыре ротора; в перспективе больше — см. этап H в таблице ниже).
- **Динамика и модули:** качество и поведение **компрессоров** (и связанной динамики) как отдельных узлов — основной звуковой фокус продукта.
- **Параллели и фаза:** выравнивание по **времени** (PDC на merge) уже есть; **фазирование / подгонка фазы** между ветками при необходимости — в зоне высокого приоритета вместе с цепочкой (не откладывать «после всего»).

**В самый конец — однозначно, после готового продукта по цепочке и динамике:**

- **Коррекция микрофона по профилю (АЧХ, EQ/FIR, измерения):** поиск характеристик, датасеты, замеры, возможные альтернативные пути — **отдельная большая работа**, не блокирует ядро.
- **Коррекция комнаты (Learn, sweep, профиль среды):** аналогично — **только после** того, как цепочка, компрессоры, параллели/фаза и UX графа доведены до состояния «продукт готов».

Идеи из блоков **2.1** и **2.2** ниже остаются **концепцией**; реализация **не в текущей очереди**.

## 1. Позиционирование

| | **ShaperX** | **Razumov Vocal Chain** |
|---|-------------|-------------------------|
| Фокус | Гибкая обработка «ваншотов», детекторы сигнала, MIDI/аудио, JSFX и т.д. | **Узкоспециализация: вокал** |
| Граф | Уже реализованы `FxGraph`, `ChainView`, Advanced canvas, сплиты/мерджи | Тот же **класс идей** (ноды, параллели, PDC), но **DSP и модули** — под вокальную цепь и фирменные компрессоры |

Идеи и паттерны из ShaperX (сериализация состояния, лента карточек, сплит/мердж, отдельный Advanced-режим) — **референс**, не слепое копирование кода.

## 2. Продуктовые цели (смысловые блоки)

### 2.1 Исправление «входа»: микрофон (отложено на финальную фазу)

- Концепция: блок **коррекции микрофона** (профили АЧХ / EQ или IR), датасет в `resources/mic_dataset/`.
- **Статус для roadmap:** реализация **после** цепочки, компрессоров, параллелей/фазы и стабильного UX; подготовка данных и замеров — **ручная/исследовательская работа**, не текущий спринт.

### 2.2 Исправление среды: комната (отложено на финальную фазу)

- Концепция: **Learn**, sweep, оценка ответа комнаты, коррекция.
- **Статус:** **после** микрофонного профиля по приоритету продукта или параллельно с ним **только** когда ядро готово; не в ближайшей очереди.

### 2.3 Фирменные компрессоры и динамика

- Модели уровня **1176, LA-2A, Tube-Tec** и т.п. — **качество** как отдельная долгая фаза (не один релиз).
- Имена в UI — как **узлы/типы**, не обязательно клоны 1:1 юридически; звук — целевой «характер». **Приоритет:** высокий, вместе с цепочкой и макросами.

### 2.4 Неограниченная серия инсертов

- **Сколько угодно** модулей подряд (два exciter, три компрессора — без искусственного лимита «как у конкурента»).
- Архитектурно: граф = список/дерево узлов + возможность **динамически** менять план (с очередью на audio thread — уже заложено в `GraphEngine`).

### 2.5 Параллельные обработки и фаза

- Параллели **популярны**; у merge уже есть **PDC** по задержкам.
- **Фаза между ветками:** при сильном фазовом сдвиге (минимум-фазовый EQ, спектральные процессы и т.д.) — **опция или автоматика** подстройки (all-pass / задержка / микро-коррекция), UX «Phase align» — **в приоритете** вместе с цепочкой и компрессорами, **не** откладывать за коррекцию микрофона.

## 3. UI / UX (целевой облик)

- **Верх:** глобальные вещи (цепочка старта, макросы; выбор профиля микрофона — когда будет DSP, сейчас не фокус).
- **Центр:** слева направо **блоки** цепочки; крупный **«+»** для добавления модуля из списка; карточки **без лишних параметров** на самой карточке.
- **Клик по блоку:** снизу **инспектор** с полным набором параметров модуля (компрессор «как положено», метры и т.д.).
- **Параллель:** от выбранного места — **ветка** (визуально вниз/в сторону), merge обратно; опционально **несколько** параллелей.
- **Advanced:** режим как **свободный граф** (ноды, кабели, браузер модулей слева) — по аналогии с ShaperX Advanced; **Main** как линейная/схематичная цепочка — проще для ежедневной работы.
- **Графика:** векторный рендер в JUCE (`paint`), токены цвета/типографики; **glow** и глубина — умеренно, с кэшем и dirty regions; **PNG** — только если для конкретного ассета выгоднее растр (фон, текстура); **SVG** — как макет/blueprint, не как единственный runtime.

---

## 4. Архитектура (слои)

1. **DSP-узлы** — модули с `getLatencySamples()`, обработка в реальном времени.
2. **План графа** — `GraphPlan` / фабрика / пользовательское дерево (serial + split + merge), смена плана **не в audio thread** без гонок.
3. **PDC** — суммарная задержка плагина, выравнивание веток на merge; расширение под фазовые темы — см. приоритеты выше.
4. **Состояние** — APVTS + сохранение; сериализация графа (версия формата).
5. **UI** — отдельно от DSP: выбор модуля → инспектор; два режима окна (Main / Advanced) при необходимости.

## 5. Крупные этапы (для следующих итераций)

Порядок **этапов в таблице** не равен порядку работ: см. раздел **«Порядок приоритетов»** в начале документа.

| Этап | Содержание | Примечание по приоритету |
|------|------------|---------------------------|
| **C. Расширяемая цепочка в UI** | Карточки + «+», список типов модулей; связь с графом. | Высокий. |
| **D. Параллели в UI** | Визуализация split/merge как в ShaperX `ChainView`. | Высокий. |
| **E. Фирменные компрессоры** | По одному типу: модель, параметры, поведение. | Высокий. |
| **G. Фаза на параллелях** | Детекция / all-pass / delay, UI «Phase align». | **Частично (v0.9.2):** целочисленный phase align на ветке split + PDC в `GraphEngine`; all-pass / UI — далее. |
| **H. Макросы ×8** | Маппинг, подписи, сохранение в состоянии. | Средний / по мере готовности цепочки. |
| **F. Advanced graph** | Canvas, кабели, браузер. | После стабильного Main. |
| **A. Профиль микрофона в DSP** | Таблицы из `mic_dataset`, EQ/FIR в `MicCorrectionNode`. | **Только в конце**, после ядра. |
| **B. Комната (Learn + sweep)** | Измерительный пайплайн, UX, профиль комнаты. | **Только в конце**, после ядра. |

Техническая микро-очередь по фазам 0–6 остаётся в **`docs/ROADMAP.md`**.

## 7. Для продолжения агентом

1. Прочитать этот файл (особенно **«Порядок приоритетов»**) и **`docs/ROADMAP.md`**.
2. Не смешивать цели Vocal Chain с задачами ShaperX 1:1 — переносить **идеи и паттерны**, не весь стек.
3. **Не** ставить в ближайшую очередь этапы **A** (микрофон) и **B** (комната), пока постановщик явно не сменит приоритет.
4. Мелкие правки — по чеклисту в roadmap; крупные DSP/UI шаги — согласовать с таблицей этапов и блоком приоритетов выше.
