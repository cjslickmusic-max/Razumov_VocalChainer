## Фаза 1 — Граф обработки (ядро продукта)

| + | # | Задача | Критерий готовности |
|---|-----|--------|---------------------|
| + | 1.1 | Модель графа: узлы, рёбра, топологическая обработка | Фиксированная цепочка из N узлов в реальном времени |
| + | 1.2 | Параллельные ветки + узел суммирования (merge) | Две ветки → один выход без слышимых артефактов при одинаковых задержках |
| + | 1.3 | **Latency:** каждый узел сообщает задержку в сэмплах; на merge — выравнивание | Юнит-тесты: известные задержки, импульс/DC |
| + | 1.4 | Очередь изменений графа (message thread → audio): без модификации вектора узлов в `processBlock` | Нет крашей при смене цепочки в хосте |
| + | 1.5 | Заглушки: `GainNode`, `FilterNode` (минимум) | Регрессия: проход сигнала, измеримая фильтрация |

**Дальше по фазе 1 (не блокер для фазы 2):** редактор графа в UI, расширенный формат плана, DSP-тесты импульса/фазы на merge.

## Фаза 2 — Параметры и состояние

| + | # | Задача | Критерий готовности |
|---|-----|--------|---------------------|
| + | 2.1 | `AudioProcessorValueTreeState`, ID параметров | Автоматизация хоста работает |
| + | 2.2 | Сохранение/загрузка пресета (XML/ValueTree) | Состояние восстанавливается после re-open проекта |
| + | 2.3 | Сглаживание параметров на DSP | Без щелчков при движении ручек |

**Реализация (v0.3.0):** `params/ParamIDs.h`, `ParameterLayout`, APVTS в процессоре; `getStateInformation` / `setStateInformation` через XML; Gain — `LinearSmoothedValue`; Filter — сглаживание частоты (экспоненциально по блоку, ~50 ms). Привязка к узлам: `GraphEngine` + gain/LP на блок (в v0.3.0 — `applyLiveParameters`).

**Реализация (v0.4.0):** цепочка `makeDefaultVocalChainPhase3` (mic-слот → gain → LP → de-esser → Opto/FET/VCA → exciter → spectral stub); APVTS на все модули; `applyPhase3Parameters` + `refreshParameterBindings`; UI — роторы и bypass.

## Фаза 3 — Модули под вокал (по приоритету)

| + | # | Задача | Критерий готовности |
|---|-----|--------|---------------------|
| + | 3.1 | **Mic slot** (первый блок по смыслу; коррекция по профилю — расширение) | Узел в цепочке и в UI, APVTS, **pass-through**; задержка 0 зафиксирована в коде |
| + | 3.2 | Компрессоры **разных типов** отдельными узлами (opto / FET / VCA-стиль…) | Разные передаточные и тайминги, не один «универсальный» морф |
| + | 3.3 | De-esser | Рабочий диапазон для типичного вокала |
| + | 3.4 | Exciter / гармоники (аккуратно, без лишнего шума) | Субъективный A/B + уровень |
| + | 3.5 | Spectral / динамический спектральный компрессор («полный спектр») | STFT + динамика по модулю с сохранением фазы; `getLatencySamples` = 1024; dry задержан (`MergeDelayPad`) + dry/wet; хост PDC; при bypass latency 0 |

*Пояснение:* **3.1** — блок остаётся; DSP коррекции по профилю (EQ/IR и т.д.) — **отдельная доработка** поверх того же узла. **3.5** (v0.8.1): Hann STFT 1024 / hop 512, компрессия по бинам в dB, фаза входа сохраняется; параллель **внутри узла** (dry/wet), не отдельный `ParallelStep` в фабрике — при необходимости граф с двумя ветками уже поддерживается `GraphEngine::processParallel` + PDC.

*Открытый хвост (вне колонки +):* **профиль микрофона** (EQ/IR/измерения); опционально сглаживание огибающей по бинам и отдельный параллельный `ParallelStep` «сухая ветка / спектральная ветка» в фабрике.

## Фаза 4 — Пресеты и продукт

| + | # | Задача | Критерий готовности |
|---|-----|--------|---------------------|
| + | 4.1 | Формат фабричных пресетов (банки) | Список имен, загрузка из ресурсов |
| + | 4.2 | «Знаковые» цепочки (в духе LA-2A / 1176 / Distressor — **как отдельные узлы**, не обязательно клоны) | Быстрый выбор стартовой цепи |
| + | 4.3 | Макросы (опционально, после стабильного графа) | N макросов → параметры узлов |

**Реализация (v0.5.0):** банк фабричных пресетов в `presets/FactoryPresets.*` (значения в коде; при необходимости тот же набор можно вынести в XML + `BinaryData`). Список имён и применение к APVTS; `getNumPrograms` / `setCurrentProgram` / `getProgramName`; в UI — выпадающий список «Preset».

**Реализация (v0.6.0):** параметр APVTS `chainProfile` (`AudioParameterChoice`): Full (Opto→FET→VCA), Compact (один Opto), FET-forward (FET→Opto→VCA). `GraphPlanFactory::makeStartupChainForIndex`, `submitPlan` при смене/загрузке состояния; в UI — «Chain» рядом с пресетом. *Compact:* в графе нет de-esser / FET / VCA — соответствующие ручки APVTS не влияют на DSP до смены цепочки.

**Реализация (v0.7.0):** четыре макроса 0…1 (нейтраль 0.5): **Glue** (makeup Opto/FET/VCA), **Air** (LP + exciter mix), **Sibilance** (порог de-esser), **Presence** (drive exciter + gain). Модуляция в `buildPhase3RealtimeParams`; роторы в UI; пресеты сбрасывают макросы в 0.5.

## Порядок следования

Строго: **0 → 1 → 2**, затем **3** параллельно с **4** (пресеты можно начинать после 2.2), **5** непрерывно после появления DSP.

## Версионирование до 1.0.0

С линейки **0.8.x** номер повышаем только «третьей цифрой»: **0.8.1**, **0.8.2**, **0.8.3** … — по одному шагу на каждую небольшую доработку или исправление. **1.0.0** зарезервирован под первый целевой продуктовый релиз (не смешиваем с промежуточными 0.8.x).
