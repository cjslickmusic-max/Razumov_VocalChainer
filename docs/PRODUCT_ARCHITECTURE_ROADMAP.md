# Razumov Vocal Chain — продуктовое видение и архитектурный roadmap

Документ для **преемственности**: цели продукта, отличие от ShaperX, крупные этапы по DSP/UI/архитектуре. Детальная табличная очередь по-прежнему в **`docs/ROADMAP.md`**.

**Визуальный референс UI:** мокапы «Main / Advanced» (лента нод + инспектор снизу; опционально граф с браузером модулей и кабелями) — стиль boutique / тёмная студия; цветовые акценты по типу обработки (см. также `docs/design/MainView_blueprint.svg`).

---

## 1. Позиционирование

| | **ShaperX** | **Razumov Vocal Chain** |
|---|-------------|-------------------------|
| Фокус | Гибкая обработка «ваншотов», детекторы сигнала, MIDI/аудио, JSFX и т.д. | **Узкоспециализация: вокал** |
| Граф | Уже реализованы `FxGraph`, `ChainView`, Advanced canvas, сплиты/мерджи | Тот же **класс идей** (ноды, параллели, PDC), но **DSP и модули** — под вокальную цепь и фирменные компрессоры |

Идеи и паттерны из ShaperX (сериализация состояния, лента карточек, сплит/мердж, отдельный Advanced-режим) — **референс**, не слепое копирование кода.

---

## 2. Продуктовые цели (смысловые блоки)

### 2.1 Исправление «входа»: микрофон

- Блок **коррекции микрофона** (профили АЧХ / EQ или дальше IR — по roadmap).
- Синтетический/реальный датасет в `resources/mic_dataset/` — этап наполнения данными.
- Цель: пресеты ближе к «одинаково хорошо» на разном железе (в разумных пределах).

### 2.2 Исправление среды: комната (второй блок, концепция)

- Пользовательский сценарий: **Learn**, микрофон включён, с колонок воспроизводится **sweep** (или иной зондирующий сигнал), плагин оценивает ответ комнаты и строит **коррекцию**.
- Учёт уже применённой **коррекции микрофона** в идеале **внутри пайплайна** (порядок и математика — отдельная DSP-задача).
- Ограничение ожиданий: комната **не решается полностью** одним плагином; цель — **улучшение**, не магия.

### 2.3 Фирменные компрессоры и динамика

- Модели уровня **1176, LA-2A, Tube-Tec** и т.п. — **качество** как отдельная долгая фаза (не один релиз).
- Имена в UI — как **узлы/типы**, не обязательно клоны 1:1 юридически; звук — целевой «характер».

### 2.4 Неограниченная серия инсертов

- **Сколько угодно** модулей подряд (два exciter, три компрессора — без искусственного лимита «как у конкурента»).
- Архитектурно: граф = список/дерево узлов + возможность **динамически** менять план (с очередью на audio thread — уже заложено в `GraphEngine`).

### 2.5 Параллельные обработки и фаза

- Параллели **популярны**; у merge уже есть **PDC** по задержкам.
- **Дополнительно (концепция):** если ветка вносит **сильный фазовый сдвиг** (минимум-фазовый EQ, спектральные процессы и т.д.), иметь **опцию** или **автоматику** подстройки фазы (all-pass / задержка / микро-коррекция) — **исследование + UX**, не обязательно в первой версии графа.

## 3. UI / UX (целевой облик)

- **Верх:** глобальные вещи (профиль микрофона, пресет, цепочка старта, макросы — в перспективе **8** роторов как главные «ручки»).
- **Центр:** слева направо **блоки** цепочки; крупный **«+»** для добавления модуля из списка; карточки **без лишних параметров** на самой карточке.
- **Клик по блоку:** снизу **инспектор** с полным набором параметров модуля (компрессор «как положено», метры и т.д.).
- **Параллель:** от выбранного места — **ветка** (визуально вниз/в сторону), merge обратно; опционально **несколько** параллелей.
- **Advanced:** режим как **свободный граф** (ноды, кабели, браузер модулей слева) — по аналогии с ShaperX Advanced; **Main** как линейная/схематичная цепочка — проще для ежедневной работы.
- **Графика:** векторный рендер в JUCE (`paint`), токены цвета/типографики; **glow** и глубина — умеренно, с кэшем и dirty regions; **PNG** — только если для конкретного ассета выгоднее растр (фон, текстура); **SVG** — как макет/blueprint, не как единственный runtime.

---

## 4. Архитектура (слои)

1. **DSP-узлы** — модули с `getLatencySamples()`, обработка в реальном времени.
2. **План графа** — `GraphPlan` / фабрика / в будущем **пользовательское дерево** (serial + split + merge), смена плана **не в audio thread** без гонок.
3. **PDC** — суммарная задержка плагина, выравнивание веток на merge; расширение под фазовые темы — отдельно.
4. **Состояние** — APVTS + сохранение; в перспективе **сериализация графа** (версия формата, как дисциплина в ShaperX).
5. **UI** — отдельно от DSP: выбор модуля → инспектор; два режима окна (Main / Advanced) при необходимости.

## 5. Крупные этапы (для следующих итераций)

Порядок может уточняться; **не все** пункты независимы.

| Этап | Содержание |
|------|------------|
| **A. Профиль микрофона в DSP** | Загрузка таблиц из `mic_dataset`, EQ/FIR в `MicCorrectionNode`, выбор профиля в UI. |
| **B. Комната (Learn + sweep)** | Измерительный пайплайн, UX, безопасность уровней, сохранение профиля комнаты. |
| **C. Расширяемая цепочка в UI** | Карточки + «+», список типов модулей; связь с графом (пока ограниченный набор типов в DSP). |
| **D. Параллели в UI** | визуализация split/merge как в ShaperX `ChainView`; опционально полная параллель в фабрике. |
| **E. Фирменные компрессоры** | По одному типу: модель, параметры, пресеты. |
| **F. Advanced graph** | Canvas, кабели, браузер, синхронизация с тем же графом. |
| **G. Фаза / параллель «под капотом»** | Исследование: детекция проблемы, all-pass / delay, UI «Phase align». |
| **H. Макросы ×8** | Маппинг на несколько параметров, подписи, сохранение в пресете. |

Техническая микро-очередь по фазам 0–6 остаётся в **`docs/ROADMAP.md`**.
